web문서와 html언어 기초

web문서의 개념과 주의사항

브라우저에서 해석하고 출력할 수 있는 모든 문서(document)를 의미함

  • 시멘틱 마크업을 지향한다
  • 웹표준 : w3c에서 제시한 문법을 준수
  • 웹접근성 : 모든 사람이 모든 환경에서 웹을 사용할 수 있도록 웹문서를 제작해야 함
  • 크로스브라우징 : 5대 브라우저에서 동일한 결과 화면과 내용을 출력할 수 있어야 함
    5대 브라우저 : IE, CHROME, SAFARI, FIREFOX, OPERA

html언어의 개념

웹문서의 기본 구조를 구축하는 언어임
(Hyper Text Markup Language)

특징과 주의사항

  • 웹문서의 구조(DOM)는 요소로 구성된다
  • 요소(element)는 tag와 속성(attribute)로 구성된다
  • 요소간의 노드(node)가 존재한다
    부모(parents), 자식(children), 자손(decendant), 형제(siblings)
  • 노드의 구조는 정해져 있다

다양한 시멘틱 요소

  • h1~h6 : 제목을 의미함, 모든 웹문서에는 h1을 하나 이상 포함해야 함(한개만 사용할 것을 권장함)
    h1~h6까지 단계적으로 사용할 것을 권장함
  • p : 관련된 하나의 문장(구, 절, 단어 두개 이상) 범위를 의미함
  • ul, ol, li : 목록을 의미함
  • strong, em, b : 강조, 중요함을 의미함
  • i : 구분의 기능
  • small : 보조적 의미
    ex) 애플(apple)사에서 아이폰을 출시했어요
  • address : 주소, 연락처, 이름 등 정보
  • blockquote, q : 인용
  • abbr : 약어
    ex) HTML hyper text markup langauge 입니다
  • kbd : 키보드
    ex) 저장 단축키는 ctrl + s 입니다
  • time : 시간
  • div, span : 의미가 없다는 의미

이스케이핑 문자

  • < : &lt;로 처리합니다
  • > : &gt;로 처리합니다
  • 여   백 : &nbsp;로 처리합니다
  • & : &amp;로 처리합니다
